# دليل لقطات واجهة الويب

هذا المجلد مخصص لحفظ الصور النهائية التي ستدرج في التقرير الأكاديمي.

## المسار

```text
report_assets/web_screenshots/
```

## الرابط المستخدم

بعد تشغيل الـ backend:

```text
http://localhost:5001/dashboard
```

## اللقطات المطلوبة

### 1. الواجهة الرئيسية

- اسم الملف: `01_dashboard_overview.png`
- المحتوى المطلوب:
  - عنوان المنصة
  - حالة الخادم
  - حالة النموذج
  - بطاقات الملخص

### 2. شاشة الحساب

- اسم الملف: `02_calculation_form.png`
- المحتوى المطلوب:
  - نموذج الإدخال
  - نوع العينة
  - درجة الحرارة
  - pH
  - NaCl
  - الوسط

### 3. نتيجة الحساب

- اسم الملف: `03_calculation_result.png`
- قبل الالتقاط أدخل:
  - `API 5L X65`
  - `25`
  - `7`
  - `3.5`
  - `seawater`
- المحتوى المطلوب:
  - `mm/yr`
  - `mpy`
  - المعادلة المستخدمة
  - معلومات النموذج
  - `R²`
  - `RMSE`
  - `MAE`

### 4. رفع ملف CSV

- اسم الملف: `04_csv_upload.png`
- المحتوى المطلوب:
  - قسم رفع الملفات
  - اختيار الملف
  - زر الرفع
  - رسالة النجاح إن أمكن

### 5. إدارة البيانات

- اسم الملف: `05_data_management.png`
- المحتوى المطلوب:
  - قسم إدارة البيانات
  - زر `مسح قاعدة البيانات`

### 6. شاشة الإحصائيات

- اسم الملف: `06_statistics_charts.png`
- المحتوى المطلوب:
  - مخطط `pH`
  - مخطط درجة الحرارة
  - مخطط الوسط

### 7. شاشة البيانات

- اسم الملف: `07_samples_table.png`
- المحتوى المطلوب:
  - جدول البيانات
  - الفلاتر
  - بعض الصفوف الظاهرة

### 8. حالة إعادة التدريب

- اسم الملف: `08_model_retrain.png`
- المحتوى المطلوب:
  - زر `إعادة تدريب النموذج`
  - ظهور طريقة الملاءمة
  - النموذج المختار

## توصية للتنسيق داخل التقرير

- استخدم عرضاً موحداً للصور
- اجعل كل صورة بعرض بين 14 و 16 سم داخل Word
- أضف عنواناً أسفل كل صورة بالشكل الآتي:

```text
شكل (1): الواجهة الرئيسية لمنصة الويب
شكل (2): نموذج إدخال بيانات حساب معدل التآكل
شكل (3): نتيجة الحساب باستخدام النموذج التنبؤي
```

## ملاحظة

إذا التقطت الصور داخل هذا المجلد بنفس الأسماء المذكورة أعلاه، سيكون من السهل جداً ترتيبها لاحقاً داخل التقرير أو إنشاء ملحق صور منظم.
